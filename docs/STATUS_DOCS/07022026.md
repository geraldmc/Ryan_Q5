### Data Files - Main 
- **TPM_filtered.csv** is unique — unlike the z-scored matrices (which are duplicated in pheno_prediction/data/), the raw filtered matrix exists only here under desync_gene_expr/. So for the MAD5000 recipe, this is the one and only source path.
<br>
<br>

The other files, for orientation:
- **TPM_master.csv** — 32,753 genes × 70 (the pre-filter superset, before dropping low-expression genes and the 2 excluded plants)
<br>

- **TPM_filtered.csv** — 19,283 × 68 ← your MAD source
<br>

- **TPM_z_scored.csv** — the z-scored version of the filtered matrix (wrong for MAD, right for modeling input)
<br>

- **TPM_pseudotime.csv**, TPM_z_scored_only_regs.csv — pseudotime-ordered and regulator-subset variants

- 1. TreeSHAP 
Designed specifically for tree-based models like XGBoost, LightGBM, and Random Forests.  TreeSHAP calculates exact Shapley values much faster than other methods by taking advantage of the tree structure.
<br>
<br>

- 2. KernelSHAP 
A model-agnostic (universal) explainer that works with any machine learning model. It works by using a weighted linear regression to estimate Shapley values. While incredibly flexible, it is computationally expensive and can be very slow with large datasets.
<br>
<br>

- 3. DeepSHAP 
Tailored for deep learning networks (e.g., TensorFlow and Keras). DeepSHAP approximates SHAP values by propagating Shapley values through the network's layers, combining this with ideas from the DeepLIFT method.4. LinearSHAPOptimized for linear and generalized additive models. It calculates exact SHAP values extremely fast, particularly when input features are assumed to be independent.
